# FinanceFlow - 스마트한 가계부 App (과제 5 고도화 결과물)

> 과제명: 리액트를 이용해서 CRUD 앱 만들기  
> 사용 방법: 이 파일은 과제 요구사항을 충족하고 한 단계 고도화한 프로젝트의 상세 안내서입니다.

## 3. 주요 기능

### 3-1. 필수 기능

| 번호 | 기능 | 구현 여부 | 설명 |
|---:|---|---|---|
| 1 | 목록 조회 Read | ■ 완료 | 수입(초록) 및 지출(빨강) 카드가 대시보드와 함께 출력됩니다. |
| 2 | 항목 추가 Create | ■ 완료 | 입력 검증을 거쳐 새로운 거래 항목을 리스트 맨 앞에 추가합니다. |
| 3 | 항목 수정 Update | ■ 완료 | 특정 거래 카드의 수정 아이콘 클릭 시 기존 값을 불러와 업데이트합니다. |
| 4 | 항목 삭제 Delete | ■ 완료 | 오버레이 컨펌창으로 의사 재확인 후 항목을 목록에서 제거합니다. |
| 5 | 데이터 구조 정의 | ■ 완료 | ID, 내역, 금액, 날짜, 메모, 타입, 생성/수정시간 데이터를 정의했습니다. |

### 3-2. 권장 기능

| 번호 | 기능 | 구현 여부 | 설명 |
|---:|---|---|---|
| 1 | 컴포넌트 분리 | ■ 완료 | App, TransactionForm, TransactionList, TransactionCard, FinanceInsights 로 분리 |
| 2 | 입력 검증 | ■ 완료 | 내역명 글자수(25자), 금액 0원 이하 검증, 날짜 필수 검증 적용 |
| 3 | 빈 상태 처리 | ■ 완료 | 내역이 아예 없거나 검색 결과가 없을 때 일러스트/설명 카드를 보여줍니다. |
| 4 | 오류 상태 처리 | ■ 완료 | 잘못된 값 입력 시 실시간 붉은색 보더와 경고 텍스트 에러를 표시합니다. |
| 5 | LocalStorage 저장 | ■ 완료 | 거래 변경 시 자동 저장 및 로드되며, 최초 사용 시 샘플 데이터를 세팅합니다. |
| 6 | 검색 또는 필터 | ■ 완료 | 키워드 검색, 타입(수입/지출/전체) 필터, 카테고리 필터, 다중 기준 정렬 지원 |

### 3-3. 도전 및 고도화 기능

| 번호 | 기능 | 구현 여부 | 설명 |
|---:|---|---|---|
| 1 | 소비 분석 리포트 | ■ 완료 | 실시간 수입 대비 지출 비율을 계산해 진행 게이지로 시각화합니다. |
| 2 | 카테고리 지출 차트 | ■ 완료 | 누적 지출이 큰 상위 카테고리 3개를 산출해 비율 차트로 보여줍니다. |
| 3 | 동적 진단 피드백 | ■ 완료 | 적자/안정/절약 상태 등 소비 비율에 맞는 맞춤형 경제 팁을 제안합니다. |
| 4 | 보안 주의사항 기록 | ■ 완료 | LocalStorage에 금융 민감 데이터 저장 금지 및 Key 관리 규칙을 명시했습니다. |

### 5-1. 설치 및 실행

1. 의존성 라이브러리를 설치합니다.

## 6. 데이터 구조

| 필드명 | 자료형 | 필수 여부 | 설명 | 예시 |
|---|---|:---:|---|---|
| id | string | 필수 | 거래 내역 항목의 고유 식별값 | `"mock-1"` 또는 `"1719234567890"` |
| type | string | 필수 | 거래 구분 (지출: `"expense"`, 수입: `"income"`) | `"expense"` |
| title | string | 필수 | 거래 내역 이름 (25자 제한) | `"스타벅스 아메리카노"` |
| amount | number | 필수 | 지출 또는 수입 액수 (1억 이하 양수) | `4500` |
| date | string | 필수 | 거래 발생 일자 (YYYY-MM-DD) | `"2026-07-09"` |
| category | string | 필수 | 수입/지출 세부 카테고리 | `"food"`, `"salary"`, `"shopping"` 등 |
| notes | string | 선택 | 사용자 메모 내용 | `"팀원들과 커피 타임"` |
| createdAt | string | 필수 | 최초 생성 일시 (ISO String) | `"2026-07-09T10:40:00.000Z"` |
| updatedAt | string | 필수 | 최종 수정 일시 (ISO String) | `"2026-07-09T10:45:00.000Z"` |

```text
실제 사용자의 금융 개인정보나 민감 데이터가 아닌 샘플 가상 데이터를 사용했습니다.
앱 로드 시 가상의 급여 및 커피 결제 등 샘플 데이터 4개가 기본 제공되어 기능을 쉽게 살펴볼 수 있습니다.
```

| 과도한 구현 | ■ 통과 | 과제 기준 이상의 복잡한 전역 상태 도구(Redux) 등을 사용하지 않고 React 훅만 활용 |

## 10. 오류 해결 기록

```text
이번 과제에서는 실행 중 치명적인 오류가 발생하지 않았습니다. 다만, 최초 Vite 프로젝트 생성 시 기존 파일이 있는 빈 상태가 아닌 디렉토리에 설치를 시도하여 경고가 발생했던 점을 파악했고, 임시 디렉토리를 경유해 안전하게 코드를 복사하는 방식으로 우회하여 해결했습니다.
```

## 16. 배운 점

1. **상태 관리의 불변성**: React에서 상태를 갱신할 때 `push()`나 `splice()` 같은 원본 파괴 메서드 대신, `map()`, `filter()`, 그리고 스프레드 연산자(`...`)를 활용하여 새 객체/배열을 복사해 주입하는 상태 업데이트 흐름을 정확히 체득했습니다.
2. **효율적인 컴포넌트Props 흐름**: 부모인 `App.jsx`에서 상태를 보유하고, 자식들이 필요한 속성과 콜백 이벤트 함수들을 단방향(One-way) Props로 전달받는 구조를 통해 유지보수가 용이해진다는 점을 깨달았습니다.
3. **UI/UX 디테일의 중요성**: 내역이 비어있을 때 적절한 일러스트/메시지를 띄워주고, 지우기 전 컨펌 모달을 마련함으로써 사용자가 실수로 데이터를 날리는 예외 상황을 설계하는 경험을 쌓았습니다.
